### Class 5 Batch and Momentum
- Because of **Parallel Computing**,Smaller batch requires longer time for one epoch (longer time for seeing all data once).
- Smaller batch size has better performance

#### Small Batch v.s. Large Batch

|                                     | Small  | Large               |
| ----------------------------------- | ------ | ------------------- |
| Speed for one update(no parallel)   | Faster | Slower              |
| Speed for one update(with parallel) | Same   | Same(not too large) |
| Time for one epoch                  | Slower | Faster              |
| Gradient                            | Noisy  | Stable              |
| Optimization                        | Better | Worse               |
| Generalization                      | Better | Worse               |

$\implies$Batch size is a hyperparameter you have to decide.

### Class 6 Adaptive Learning Rate
![](screen%20shot/Pasted%20image%2020251110170834.png)
Training can be difficult even without critical points.
 ——**Learning rate cannot be one-size-fits-all**
 ![](screen%20shot/Pasted%20image%2020251110171422.png)
Root Mean Square

$\theta_i^{1} \leftarrow \theta_i^{0} - \frac{\eta}{\sigma_i^{0}} g_i^{0}$                   $\sigma_i^{0} = \sqrt{(g_i^{0})^{2}} = |g_i^{0}|$
$\theta_i^{2} \leftarrow \theta_i^{1} - \frac{\eta}{\sigma_i^{1}} g_i^{1}$                   $\sigma_i^{1} = \sqrt{\frac{1}{2}\left[(g_i^{0})^{2} + (g_i^{1})^{2}\right]}$
$\theta_i^{3} \leftarrow \theta_i^{2} - \frac{\eta}{\sigma_i^{2}} g_i^{2}$                   $\sigma_i^{2} =\sqrt{\frac{1}{3}\left[(g_i^{0})^{2} + (g_i^{1})^{2} + (g_i^{2})^{2}\right]}$
$\vdots$
$\theta_i^{t+1} \leftarrow \theta_i^{t} - \frac{\eta}{\sigma_i^{t}} g_i^{t}$                 $\sigma_i^{t} =\sqrt{\frac{1}{t+1}\sum_{i=0}^{t}(g_i^{t})^{2}}$

RMSProp                           (0<$\alpha$<1)
$\theta_i^{1} \leftarrow \theta_i^{0} - \frac{\eta}{\sigma_i^{0}} g_i^{0}$                   $\sigma_i^{0} = \sqrt{(g_i^{0})^{2}} = |g_i^{0}|$
$\theta_i^{2} \leftarrow \theta_i^{1} - \frac{\eta}{\sigma_i^{1}} g_i^{1}$                   $\sigma_i^{1} = \sqrt{\left[\alpha(\sigma_i^{0})^{2} + (1-\alpha)(g_i^{1})^{2}\right]}$
$\theta_i^{3} \leftarrow \theta_i^{2} - \frac{\eta}{\sigma_i^{2}} g_i^{2}$                   $\sigma_i^{2} = \sqrt{\left[\alpha(\sigma_i^{1})^{2} + (1-\alpha)(g_i^{2})^{2}\right]}$
$\vdots$
$\theta_i^{t+1} \leftarrow \theta_i^{t} - \frac{\eta}{\sigma_i^{t}} g_i^{t}$                  $\sigma_i^{t} = \sqrt{\left[\alpha(\sigma_i^{t-1})^{2} + (1-\alpha)(g_i^{t})^{2}\right]}$
——The recent gradient has larger influence, and the past gradients have less influence

#### Learning Rate Scheduling              $\theta_i^{t+1} \leftarrow \theta_i^{t} - \frac{\eta^t}{\sigma_i^{t}} g_i^{t}$  
Learning Rate Decay 
As the training goes, we are closer to the destination, so we reduce the learning rate.

Warm Up 
Increase and then decrease
At the beginning, the estimate of $\sigma_i^{t}$ has large variance.

### Class 7 Classification 
#### Classification as Regression
Class as one-hot vector(the vector that only contain 0 and 1) 
![](screen%20shot/Pasted%20image%2020251112205645.png)

Soft-max 
input: logit

$$
y_i' = \frac{e^{y_i}}{\sum_j e^{y_j}}
\quad
\begin{aligned}
1 > y_i' > 0 \\
\sum_i y_i' = 1
\end{aligned}
$$
- widen the difference between bigger and smaller numbers
- when there's only 2 classes, soft-max is equivalent to sigmoid

Loss of Classification 
Mean Square Error (MSE):   $e = \sum_i(\hat{y_i}-y_i')^2$
Cross-entropy(better):   $e=-\sum_i\hat{y_i}lny_i'$
Minimizing cross-entropy is equivalent to maximizing likelihood.
Why Cross-entropy is better: Can work even when the gradient is very small

Changing the loss function can change the difficulty of optimization.

Key Matrix ($K$)
$$\boldsymbol{k}^i = W^k\boldsymbol{a}^i$$
$$\underbrace{\begin{bmatrix} \boldsymbol{k}^1 & \boldsymbol{k}^2 & \boldsymbol{k}^3 & \boldsymbol{k}^4 \end{bmatrix}}_{K} = W^k \underbrace{\begin{bmatrix} \boldsymbol{a}^1 & \boldsymbol{a}^2 & \boldsymbol{a}^3 & \boldsymbol{a}^4 \end{bmatrix}}_{I}$$

$$\boldsymbol{v}^i = W^v\boldsymbol{a}^i$$
$$\underbrace{\begin{bmatrix} \boldsymbol{v}^1 & \boldsymbol{v}^2 & \boldsymbol{v}^3 & \boldsymbol{v}^4 \end{bmatrix}}_{V} = W^v \underbrace{\begin{bmatrix} \boldsymbol{a}^1 & \boldsymbol{a}^2 & \boldsymbol{a}^3 & \boldsymbol{a}^4 \end{bmatrix}}_{I}$$

### Class 10 Batch Normalization（批次标准化）

#### Changing Landscape: Let different dimensions have similar range —— Feature Normalization
![](screen%20shot/机器学习20218.1.png)
variances：方差
$\tilde{x}$
注：每经过一层Neural Network都要进行一次Feature Normalization 在Activation Function前或后都可以（如果Activation Function 是Sigmoid建议在之前）
![](screen%20shot/机器学习2021%208.2.png)
- Normalization 让不同examples之间产生联系($\tilde{x^i}$变了其他都会跟着变)
- 为了避免输入data的量过大每次考虑一个 batch ( batch 的大小需要足够大)
- $\hat{z^i} = \gamma \odot \tilde{z^i}+\beta :避免mean始终为0等不必要的限制$（$\gamma 和\beta也是network的参数\, \gamma 一开始是1-vector, \beta一开始是0-vector$）
#### Batch normalization – Testing
We do not always have batch at testing stage:
- Computing the moving average of 𝝁 and 𝝈 of the batches during training.
$$\overline{\boldsymbol{\mu}}^\prime \leftarrow p \overline{\boldsymbol{\mu}} + (1 - p)\mu^{t}$$
$$
\tilde{\boldsymbol{z}} = \frac{\boldsymbol{z} - \bar{\boldsymbol{\mu}}}{\bar{\boldsymbol{\sigma}}}
$$
